Lesson 4: Basic Programming Concepts

Computer science and programming is about solving problems by breaking them into smaller pieces, and then methodically, describe a sequence of steps required to solve those problems.

Just as we said before that computers are stupid and will only do what you, a programmer, tells it to do, the instructions for telling a computer what to do is called a program. A program is a sequence of steps executed in a logical progression.

Python is a high level language where user instructions are not executed directly by the computer. Instead, instructions are input into the Python interpreter, which then translates the user’s input into an executable that the computer understands.

Writing code requires a user to understand abstract concepts, such as multiple programs working in tandem with each other. A programmer also needs to understand her goals and instructions (inputs and outputs) in a logical and procedural manner while paying great attention to detail, since a programming language is much more precise than a natural language like English, and there is no room for ambiguity. Once the problem is understood, a programmer then creates a strategy and breaks the problem down into small pieces that can be solved concisely. These small solutions then ripple back upwards to solve the larger problem.

Comments and Expressions

A line that starts with # is considered to be a comment and will be ignored by the Python interpreter.

An expression can be made up of multiple expressions and operators. Each expression can be non-terminal and be replaced by other expressions, or terminal, which cannot be replaced.

Operators can be +, *, <=, >=, !=, ==. The double equal is the equality comparison, because a single = means an assignment.

When comparing numbers, the output is a boolean value, either true or false.

Variables

Variables are names that keep track of known or unknown values, makes programs easier to understand.

They are defined with an assignment statement: name = expression. The name needs to start with a letter or an underscore. The beauty of the variable is that it can change without having to change the entire equation.

Strings

A string is a set of characters within either matching single quotes or double quotes. The quotes are necessary because without them, Python will think the string of characters is a variable.

Python does not allow concatenation of strings and variables, but allows multiplication of strings and variables.

Indexing

<string>[<expression>] – indicates the position of a character in a string. (eg. name[0])

Going forward, indexing starts with 0, but going backwards, indexing starts with -1. The front of the index is inclusive, the end of the index is exclusive.

<string>[<expression>:<expression>] – indicates indexing from start expression to (stop -1) expression.

If <expression> is left blank, Python will index it as either the beginning or end. (eg. name[:] would return the full string)

Find

The find() function in Python returns the index of the first position in the search string where the target string is found. If the search string is not found in the target string, find() returns -1.

target_string.find(search_string)

target_string.find(search_string, <number>) returns the index of the first position in the search string where the target string is found, after the <number> specified, inclusively.

Lesson 5: Functions

A function is a mechanism to do an operation or series of operations so that a user does not need to write out all the operations over and over again whenever that set of operations is needed. This avoidance of repetition can also avoid typographical errors if something within the function needs to be changed.

def (arg, arg, etc):

Define the function using def <name of function>:. Instead of using parentheses to bound the actions of the function, Python uses tab indents.

A function without a return statement will have no output.

Lesson 7: Lists and For Loops

As previously stated, a string is a sequence of characters. A list is a sequence of anything - strings, other lists, or characters.

In a string, the set of characters is surrounded by single or double quotes. In a list, the sequence of items in the list is surrounded by square brackets, with each item separated by commas. A list can be assigned to a variable. You can also access different elements of the list in the same way that you index elements in a string, using square brackets and the position of the element you’re referencing. Indexing a string returns a string. Indexing one element from a list returns the element. Indexing multiple elements from a list returns a list of those elements.

[<expression>,<expression>, etc]

Lists can be a mix of strings, numbers and lists.

Mutation and Aliasing

Lists support mutation, where we can change the value of a list after we created it, and it subsequently changes the variables that point to it. With a string, you can change the value of a variable that is assigned to a string, but you don’t ever change the string itself.

Aliasing is having 2 different ways to refer to the same element. Any change made to the object that the first variable points to also changes the value of the second variable.

Append

<list>.append(<element>) – You can append, or add, elements to a list.

len(<list>) – gives number of items in the element. (eg. len(“Udacity”) = 7)

The len function only counts the number of items in the first level. So if a list includes other lists, the inner list is counted as one item.

For Loops

<list>.index(<value>) – gives the first position index in <list> where the term <value> is found.

<value> in <list> – if <value> is in <list>, returns True. Otherwise, return False.

<value> not in <list> – if <value> is not in <list, returns True. Otherwise, return False.
